# Landing Section Visibility System — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an admin-controlled visibility system for individual landing sections (Nuestra historia, Favoritos, Promociones, Galería, Reseñas, Contacto) and CTA buttons (Reservar, Ver menú, WhatsApp flotante), with a live iframe preview that reflects unsaved changes.

**Architecture:** Eight new boolean columns in `cafe_settings` + a single `sectionFlags` state object on the admin page (matching the existing `show_promotions` pattern) + iframe-based live preview using `window.postMessage`. The landing reads an `effectiveSettings` (DB settings merged with preview overrides when running inside an iframe) and gates each section/button on its toggle.

**Tech Stack:** React 19, TypeScript 5.9, react-hook-form, shadcn/ui (adding Switch), Tailwind v4, framer-motion, Supabase Postgres.

**Spec reference:** [docs/superpowers/specs/2026-04-10-landing-section-visibility-design.md](../specs/2026-04-10-landing-section-visibility-design.md)

Below the existing `useEffect` blocks, add:

```tsx
// Live preview: when rendered inside an iframe, accept settings overrides via postMessage.
useEffect(() => {
  if (typeof window === "undefined" || window.parent === window) return;

  const handler = (e: MessageEvent) => {
    if (e.origin !== window.location.origin) return;
    if (e.data?.type === "preview-update" && e.data.settings) {
      setPreviewOverrides(e.data.settings as Partial<CafeSettings>);
    }
  };

  window.addEventListener("message", handler);
  return () => window.removeEventListener("message", handler);
}, []);
```

- [ ] **Step 6.1: Add iframeRef and the dispatcher**

Locate the `useRef` calls near the top of the component (search for `useRef`). Add:

```tsx
const previewIframeRef = useRef<HTMLIFrameElement>(null);
```

After the existing effects but before `return (` , add the dispatcher:

```tsx
useEffect(() => {
  const sendPreview = () => {
    const frame = previewIframeRef.current;
    if (!frame) return;
    const formValues = getValues();
    const payload = {
      ...settings,
      ...formValues,
      logo_url: logoUrl,
      cover_url: coverUrl,
      about_image_url: aboutImageUrl,
      gallery_urls: galleryUrls,
      show_promotions: showPromotions,
      testimonials: testimonials.map(({ _id, ...rest }) => rest),
      featured_product_ids: featuredIds,
      custom_palettes: customPalettes,
      ...sectionFlags,
    };
    frame.contentWindow?.postMessage(
      { type: "preview-update", settings: payload },
      window.location.origin,
    );
  };

  // Send once whenever any non-form state changes.
  sendPreview();

  // Subscribe to form changes (debounce-free; react-hook-form's watch already
  // fires only on actual field changes).
  const subscription = watch(() => sendPreview());
  return () => subscription.unsubscribe();
}, [
  settings,
  logoUrl,
  coverUrl,
  aboutImageUrl,
  galleryUrls,
  showPromotions,
  testimonials,
  featuredIds,
  customPalettes,
  sectionFlags,
  watch,
  getValues,
]);
```
